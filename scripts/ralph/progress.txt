# Ralph Progress Log

Started: 2026-06-20

## Codebase Patterns

- The current implementation is a dependency-free Node ESM server. `npm test` and `npm run check` are the mandatory local quality gates.
- `LABS_DEMO_MODE` and Vercel's ephemeral `/tmp` database are demo-only boundaries; production code must fail closed when essential configuration is missing.

- Removed Vercel as an implicit demo-mode switch. Demo SQLite and seeded identities now require `LABS_DEMO_MODE=true`; non-demo runtime avoids SQLite/demo fallback and remains fail-closed until the planned production adapters are available.

## 2026-06-20 — US-003 Verify OIDC bearer tokens

- Implemented `OidcIdentityProvider` with server-side bearer-token verification against explicitly configured HTTPS issuer, audience, tenant claim, and JWKS URL. The verifier accepts only `RS256` and `ES256` signing keys from the configured JWKS, caches keys briefly, refreshes once on an unknown key ID, and never logs token content.
- Routed API identity resolution asynchronously through the verifier. Invalid token syntax, signatures, expiry, issuer, and audience return non-sensitive structured authentication errors; unavailable JWKS returns a non-sensitive service-unavailable error.
- Added the `LABS_OIDC_JWKS_URL` runtime contract, HTTPS configuration validation, operator documentation, and focused cryptographic verification tests. Files changed: `src/identity-provider.mjs`, `src/runtime-config.mjs`, `server.mjs`, `test/identity-provider.test.mjs`, `test/runtime-config.test.mjs`, `README.md`, and `docs/runtime-configuration.md`.
- Verification: `npm test` (21 passing) and `npm run check` passed. Vercel CLI authentication and project linkage are available, but `npx vercel env ls preview` reported `No Environment Variables found`; no preview was created because the required production configuration is absent.
- Reusable learning: mutate a significant decoded signature byte when testing JWS signature rejection; a change only to unused base64url padding bits may leave the decoded signature unchanged.

## 2026-06-20 — US-005 Add secure session and CSRF protection

- Added `src/request-security.mjs`, which preserves verified OIDC bearer-token authentication as the only current production transport and requires an exact same-origin, constant-time matched CSRF token for any future cookie-authenticated mutation. - Added targeted CSRF acceptance/rejection and response-header tests. Files changed: `src/request-security.mjs`, `src/identity-provider.mjs`, `server.mjs`, `test/request-security.test.mjs`, `package.json`, and `README.md`.

- Reusable learning: treat CSRF as a property of cookie credentials, not a blanket replacement for bearer-token verification; apply the same CSP and security headers to JSON error responses as to static assets.

## 2026-06-20 — US-007 Add PostgreSQL migration framework

- Added PostgreSQL core tables for cycles, projects, gates, evidence, reviews, decisions, approvals, handoffs, and audit events.
