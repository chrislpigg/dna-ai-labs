# Ralph Progress Log

## Codebase Patterns

- The current implementation is a dependency-free Node ESM server. `npm test` and `npm run check` are the mandatory local quality gates.
- `LABS_DEMO_MODE` and Vercel's ephemeral `/tmp` database are demo-only boundaries; production code must fail closed when essential configuration is missing.

## 2026-06-20 — US-001 Document production runtime configuration

- Implemented an explicit runtime configuration contract in `src/runtime-config.mjs`; production validates OIDC, PostgreSQL, tenant, approved-artifact, notification, and integration provider configuration using non-secret issue codes.
- Removed Vercel as an implicit demo-mode switch. Demo SQLite and seeded identities now require `LABS_DEMO_MODE=true`; non-demo runtime avoids SQLite/demo fallback and remains fail-closed until the planned production adapters are available.
- Added `docs/runtime-configuration.md`, README deployment guidance, focused configuration tests, and syntax coverage for the new module. Files committed in `22491ea` with `feat: US-001 - Document production runtime configuration`.

- Reusable learning: do not infer demo mode from a deployment platform. Treat demo mode as an explicit operator decision, and expose stable missing-configuration codes rather than environment values.

## 2026-06-20 — US-002 Add OIDC identity-provider interface
